# 📘 GreatDocuments — плагин паспортной системы для Minecraft

GreatDocuments — это плагин для серверов Minecraft (Bukkit/Spigot/Paper), добавляющий внутренние и заграничные паспорта в виде книг с возможностью просмотра, редактирования и запроса на просмотр. Все данные хранятся в конфигурационном файле и привязаны к UUID игрока.

## 🚀 Возможности

- **Внутренний паспорт гражданина** — содержит личные данные, биографию, прописку, информацию о выдаче.
- **Заграничный паспорт** — содержит основные данные, ссылку на внутренний паспорт и собственный номер.
- **Редактирование полей** — команды для изменения данных паспорта (с проверкой допустимых значений).
- **Запрос на просмотр** — возможность показать свой паспорт другому игроку на расстоянии до 15 блоков.
- **Автоматическое обновление дат** — при изменении любого поля паспорта обновляются дата выдачи, срок действия и поле «Кем выдан».
- **Цветное оформление** — книги используют компоненты Adventure с цветами и форматированием.
- **Поддержка таб-комплешена** — автодополнение для всех подкоманд и полей.

## 📦 Установка

1. Скачайте последний релиз плагина.
2. Поместите `.jar` файл в папку `plugins` вашего сервера.
3. Перезапустите сервер (или используйте `/reload` с осторожностью).
4. При первом запуске создастся `config.yml` с необходимыми секциями.

> **Требования:**  
> - Сервер на базе Paper 1.16.5+ (или Spigot с поддержкой Adventure).  
> - Java 8 или выше.

## 🔧 Команды

Все команды используют основной алиас `/gd`.

| Подкоманда | Аргументы | Описание | Разрешение |
|------------|-----------|----------|-------------|
| `passport` | нет | Открыть свой внутренний паспорт | все |
| `foreignpassport` | нет | Открыть свой заграничный паспорт | все |
| `view` | `<игрок>` | Отправить запрос на просмотр паспорта игроку (дистанция ≤15 блоков, кулдаун 30 сек) | все |
| `acceptview` | `<игрок>` | Принять запрос на просмотр от указанного игрока | все |
| `denyview` | `<игрок>` | Отклонить запрос | все |
| `setpassport` | `[игрок] <поле> <значение>` | Установить поле внутреннего паспорта (если игрок не указан — себе) | `greatdocuments.setpassport` |
| `setforeignpassport` | `[игрок] <поле> <значение>` | Установить поле заграничного паспорта | `greatdocuments.setforeignpassport` |

### Поля для `setpassport`

| Поле (русское) | Описание | Допустимые значения / проверки |
|----------------|----------|--------------------------------|
| `гражданство` | Гражданство | текст |
| `возраст` | Возраст | целое число |
| `государство` | Государство (паспортная принадлежность) | текст |
| `номер` | Номер паспорта | текст |
| `пол` | Пол | `Мужской` или `Женский` |
| `рост` | Рост (см) | целое число |
| `месторождения` | Место рождения | текст |
| `семейноеположение` | Семейное положение | `Женат`, `Не женат`, `Замужем`, `Не замужем` |
| `супруг` | Имя супруга/супруги | текст |
| `страна` | Страна прописки | текст |
| `район` | Район прописки | текст |
| `улица` | Улица прописки | текст |
| `дом` | Номер дома | текст |
| `координаты` | Координаты места жительства | текст |
| `министерство` | Министерство, выдавшее паспорт | текст |
| `должность` | Должность выдавшего | текст |

### Поля для `setforeignpassport`

| Поле | Описание | Проверки |
|------|----------|----------|
| `гражданство` | Гражданство | текст |
| `возраст` | Возраст | целое число |
| `пол` | Пол | `Мужской`/`Женский` |
| `номер` | Номер заграничного паспорта | текст |

> **Примечание:** При редактировании любого поля внутреннего паспорта автоматически обновляются даты (`issueDate`, `validUntil`) и поле `issuedBy` (ставится имя редактора). Для заграничного паспорта срок действия — 2 недели, для внутреннего — 1 месяц.

## 🔐 Разрешения

| Разрешение | Описание |
|------------|----------|
| `greatdocuments.setpassport` | Разрешает использовать `/gd setpassport` |
| `greatdocuments.setforeignpassport` | Разрешает использовать `/gd setforeignpassport` |

По умолчанию никакие разрешения не выдаются (OP может всё). Выдавайте их через менеджер прав (LuckPerms и т.п.).

## ⚙️ Конфигурация

Файл `config.yml` создаётся автоматически. Структура:

```yaml
players:
  <UUID игрока>:
    # Внутренний паспорт
    passportState: "Гражданин"
    citizenship: "Россия"
    age: "25"
    gender: "Мужской"
    height: "180"
    birthplace: "Москва"
    maritalStatus: "Женат"
    spouse: "Анна"
    residence:
      country: "Россия"
      district: "Центральный"
      street: "Ленина"
      house: "15"
      coords: "x:100 y:64 z:200"
    passportNumber: "1234 567890"
    issueDate: "27.04.2026"
    validUntil: "27.05.2026"
    issuedBy: "Admin"
    ministry: "Министерство юстиции"
    issuerPosition: "Начальник отдела"
    # Заграничный паспорт
    foreignPassport:
      citizenship: "Россия"
      age: "25"
      gender: "Мужской"
      number: "AB123456"
      issueDate: "27.04.2026"
      validUntil: "11.05.2026"
      issuedBy: "Admin"
```

Все даты в формате `dd.MM.yyyy`. При установке поля через команду плагин автоматически создаёт нужные секции.

## 📖 Примеры использования

1. **Выдать себе внутренний паспорт с данными**  
   `/gd setpassport государство Гражданин`  
   `/gd setpassport гражданство Россия`  
   `/gd setpassport возраст 25`  
   `/gd setpassport пол Мужской`  
   ...  
   Затем открыть: `/gd passport`

2. **Выдать загранпаспорт игроку**  
   `/gd setforeignpassport Steve гражданство США`  
   `/gd setforeignpassport Steve возраст 30`  
   `/gd setforeignpassport Steve пол Мужской`  
   `/gd setforeignpassport Steve номер USA123456`  
   Открыть у Steve: `/gd foreignpassport`

3. **Показать свой паспорт другому игроку**  
   `/gd view Steve`  
   Steve получит сообщение:  
   *"Игрок показывает вам паспорт. [Посмотреть] [Отказаться]"*  
   При нажатии на «Посмотреть» откроется паспорт запросившего.

4. **Редактировать паспорт другого игрока (с разрешением)**  
   Подойдите к нему на расстояние ≤20 блоков и выполните:  
   `/gd setpassport Steve семейноеположение Женат`

## 🛠️ Для разработчиков

Плагин написан на Java с использованием:
- [PaperAPI / Adventure](https://docs.advntr.dev/) для текстовых компонентов.
- Bukkit API для работы с командами, книгами и конфигурацией.

Особенности реализации:
- Класс `PassportBook` — генерация внутреннего паспорта.
- `ForeignPassportBook` — заграничного.
- `DocumentUtils` — вспомогательные методы (центрирование текста, обновление дат).
- `PassportCommand` — обработка всех подкоманд и таб-комплешн.
